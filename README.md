# Deposition Prep — Client Portal
### Lexiden Legal Tech Hack · 2026

AI-powered deposition preparation for legal clients. Upload your case documents,
get a plain-language prep guide, then practice answering questions with real-time coaching.

---

## Quick start

```bash
# 1. Install
npm install

# 2. Add your API key
cp .env.example .env
# Edit .env → set VITE_ANTHROPIC_API_KEY=sk-ant-...

# 3. Run
npm run dev
# → http://localhost:5173
```

## How it works

### Step 1 — Upload
Client uploads up to 5 case documents (PDF, DOCX, TXT). Files are read client-side
using the FileReader API — nothing is sent to any server except the Anthropic API.
Client selects case type and their role (plaintiff / defendant / witness).

### Step 2 — Case summary
Claude reads all uploaded documents and returns a plain-language 3-4 sentence
summary of the case. Client confirms this is accurate before proceeding.

### Step 3 — Prep guide
Claude generates case-specific DOs, DON'Ts, and key reminders grounded in the
actual documents. Not generic advice — pulled from what is actually in their files.

### Step 4 — Simulation
6 deposition questions generated by Claude, structured as:
- Q1–2: Background (role, timeline, relationships)
- Q3–4: Core facts and claims from the documents
- Q5–6: Challenging questions targeting weak points

Client answers each question by voice (Web Speech API) or text.
After each answer, Claude evaluates:
- **Pace** — rushed / hesitant / good (measured by response time + word count)
- **Filler words** — um, uh, like, you know, etc. (detected client-side)
- **Relevance** — On Point / Off Track / Over-Explained
- **Tone** — Confident / Neutral / Defensive / Evasive
- **Volunteered info** — did they give information that wasn't asked for?
- **Coaching tip** — one plain-language sentence

### Step 5 — Report
Score out of 100, strengths, weaknesses, personalised DOs/DON'Ts based on
actual performance, full Q&A review with coach notes. Downloadable as .txt.

## Tech stack

- **React 18** + Vite
- **Web Speech API** — voice input (Chrome / Safari)
- **Claude Sonnet 4** — document analysis, question generation, answer evaluation, report
- **CSS Modules** — scoped styles, dark mode via CSS variables
- **localStorage** — session persistence (no backend required)

## Adding lawyer-side features (next phase)

The codebase is structured to add a lawyer view alongside the client view:

1. Add a `role` selector at app entry (`lawyer` | `client`)
2. Add `LawyerPhase` components under `src/components/lawyer/`
3. Add lawyer-specific Claude calls to `src/lib/claude.js`:
   - `analyzeWitnessQuestions(questions)` — improve / suggest additions
   - `generateLawyerOutline(documents)` — full depo outline per SKILL.md structure
4. For firm-level RAG: add a settings page where lawyers upload firm docs,
   embed them (e.g. via Anthropic's document API), and pass as context to all calls

## Production notes

The API key is injected client-side via Vite env vars — fine for a hackathon demo.
For production: proxy all `/v1/messages` calls through your own backend endpoint
and never expose the key in a browser bundle.

```bash
npm run build   # output in /dist — deploy to Vercel, Netlify, or any static host
```
